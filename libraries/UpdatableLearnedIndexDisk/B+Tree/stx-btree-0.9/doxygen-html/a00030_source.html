00041 /** @brief Specialized B+ tree template class implementing STL's multimap
00042  * container.
00043  *
00044  * Implements the STL multimap using a B+ tree. It can be used as a drop-in
00045  * replacement for std::multimap. Not all asymptotic time requirements are met
00046  * in theory. The class has a traits class defining B+ tree properties like
00047  * slots and self-verification. Furthermore an allocator can be specified for
00048  * tree nodes.
00049  *
00050  * Most noteworthy difference to the default red-black implementation of
00051  * std::multimap is that the B+ tree does not hold key and data pair together
00052  * in memory. Instead each B+ tree node has two arrays of keys and data
00053  * values. This design directly generates many problems in implementing the
00054  * iterator's operator's which return value_type composition pairs.
00055  */
